Mode de fonctionnement du groupe

01
Lancement et organisation

Le groupe a été constitué et le groupe WhatsApp créé le 02 juin 2026 par TOVIHOUANDE Victoria Gloria. Dès le premier jour, chaque membre s'est présenté et le groupe a analysé ensemble le cahier de charges. La première décision collective a été de configurer GitHub : Victoria a créé le dépôt, chaque membre a envoyé son nom de compte et a été invité. Le nom du dépôt PIL1_2526_10 a été adopté collectivement .

02
Communication et suivi

Les échanges quotidiens se font via le groupe WhatsApp . Les membres y partagent leurs avancées, leurs blocages et s'entraident. Victoria assure le rôle de coordinatrice : elle relance les membres, vérifie les avancements et prend les décisions techniques importantes après discussion collective. Tout au long du projet nous avons eu 3 rencontres en présentiel à la bibliothèque de la FSA du Lundi 08 Juin au Mercredi 10 juin de 09h à 17h .

03
Développement collaboratif avec Git

Chaque membre travaille sur une branche dédiée à son module. Les fichiers sont partagés entre membres lorsque nécessaire — par exemple Victoria a transmis ses fichiers d'authentification à TCHIBOZO Michel pour l'intégration du CSS commun afin de tester une possible fusion entre leurs deux branchessss . Des commits réguliers permettent de tracer l'avancement individuel de chacun.

04
Intégration et tests

La phase d'intégration a consisté à assembler les modules développés en parallèle. DJOSSOU Eustache et KAGBO Bel-ange ont coordonné la fusion du module de messagerie avec la base de données. KAGBO a également ajouté les tables manquantes à la BDD pour assurer la cohérence de l'ensemble.

05
Finalisation et livraison

Les derniers jours avant la deadline du 10 juin 2026 ont été consacrés aux corrections finales, à l'installation des dépendances (PostgreSQL notamment), à la rédaction du rapport technique et du fichier SQL, et au dépôt final de tous les livrables sur le dépôt GitHub.

Description de la conception

🏗️

Architecture

L'application suit une architecture client-serveur classique :

  • Frontend : HTML/CSS/JavaScript
  • Backend : Python avec Django
  • Base de données : PostgreSQL(géré via pgAdmin4)
📦

Modules principaux

L'application est découpée en quatres appications distictes : :

  • Gestion des comptes et profils et authentification
  • Mise en correspondance automatique (matching)
  • Messagerie instantanée
  • annonces - offres et demande de mentorat
🤝

Algorithme de matching

Le score de compatibilité est calculé en pondérant :

  • Analyse des profils utilisateurs (compétences, filière, niveau d'étude
  • Analyse des offres et demandes publiées
🔒

Sécurité

Les mesures de sécurité implémentées incluent :

  • Hachage des mots de passe via PBKDF2
  • Authentification par session Django
  • Validation côté serveur des données
  • Accès restreint aux données personnelles

Instructions de déploiement

1

Prérequis

Vérifier que les outils suivants sont installés sur la machine :

Python 3.12  ·  PostgreSQL  ·  Git  ·  VS Code (recommandé)

2

Cloner le projet

Récupérer le code source depuis GitHub.

git clone https://github.com/Victoria-tec/PIL1_2526_10.git
cd PIL1_2526_10
3

Créer et activer l'environnement virtuel

Windows :

python -m venv env
env\Scripts\activate.bat

Linux / Mac :

python3 -m venv env
source env/bin/activate
4

Installer les dépendances

Installer tous les paquets requis listés dans requirements.txt.

pip install -r requirements.txt
5

Configurer la base de données

Ouvrir pgAdmin 4 et créer une base de données nommée ifri_mentorlink. Ensuite ouvrir le fichier IFRI_MentorLink/settings.py et modifier les champs USER et PASSWORD avec vos identifiants PostgreSQL.

6

Appliquer les migrations

Créer toutes les tables dans la base de données.

python manage.py migrate
7

Lancer le serveur

Démarrer le serveur de développement Django.

python manage.py runserver
8

Ouvrir dans le navigateur

L'application est accessible à l'adresse suivante :

http://127.0.0.1:8000

Problèmes rencontrés lors du projet

Organisation

Nous avions comme idée la création de deux team : une pour gérer le backend et l'autre pour gérer le frontend.

Fusion

Nous avons, vers la fin du projet, décidé de merge toutes les branches dans une branche develop. Cela avait pour pour objectif d'avoir une idée de ce que donnerait le main. Nous avons alors rencontré plusieurs problèmes de conflits que nous avons réglés en présentiel.

Test

Nous avons rencontrer plusieurs bugs éventuels lors des tests effectués .

Manuel d'utilisation

📝 Créer un compte

  1. Accéder à la page d'accueil
  2. Cliquer sur « S'inscrire »
  3. Remplir le formulaire (nom, prénom, email, téléphone, mot de passe,sexe )
  4. Sélectionner ses points forts et points faibles
  6. Valider l'inscription

🔑 Se connecter

  1. Accéder à la page de connexion
  2. Saisir son email ou numéro de téléphone
  3. Entrer son mot de passe
  4. Cliquer sur « Connexion »
  5. En cas d'oubli, utiliser « Mot de passe oublié »

👤 Gérer son profil

  1. Accéder à « Mon Profil » depuis le menu
  2. Modifier ses informations personnelles
  3. Ajouter/modifier ses compétences et lacunes
  4. Renseigner ses disponibilités horaires
  5. Enregistrer les modifications

🤝 Publier une offre ou demande

  1. Cliquer sur le bouton créer
  2. Choisir « Offre » (je peux aider) ou « Demande » (j'ai besoin d'aide)
  3. Sélectionner la matière concernée
  4. Indiquer ses disponibilités et le format souhaité
  5. Publier l'annonce

🔍 Trouver un mentor/mentoré

  1. consulter votre tableau de bord
  2. Consulter les suggestions automatiques
  3. Filtrer par matière, filière ou disponibilité
  4. Consulter le profil et le score de compatibilité
  5. Envoyer une demande de mise en relation

💬 Utiliser la messagerie

  1. Accéder à la section « Messages »
  2. Sélectionner une conversation existante
  3. Taper le message dans la zone de saisie
  4. Envoyer avec le bouton ou la touche Entrée
